  • Next.js 13 설치


주소: Next.js 13 Install




주소를 클릭하면 나오는 페이지에서 Automatic Installation의 Terminal에 있는 명령어를 복사하고, 붙여넣습니다.



원하는 프로젝트 명] What is your project named? my-app

TypeScript 사용여부] Would you like to use TypeScript? No / Yes 

ESLint 사용여부] Would you like to use ESLint? No / Yes 

Tailwind CSS 사용여부] Would you like to use Tailwind CSS? No / Yes 

루트 디렉토리 src] Would you like to use `src/` directory? No / Yes 

App Router 또는 pages Router. (Next.js 13부터는 App Router 권장)] Would you like to use App Router? (recommended) No / Yes 

tsconfig paths alias default @/* 사용여부] Would you like to customize the default import alias (@/*)? No / Yes 

tsconfig paths alias prefix 지정] What import alias would you like configured? @/*


위와 같이 작업이 끝났다면 Next.js 13이 모두 설치된 상태입니다.




  • Github repository 생성


1. New를 클릭해서 새로운 repository를 생성합니다.




2. 생성할 때, Public으로 합니다. 

Private으로 하면 깃헙블로그는 유료버전을 사용해야 합니다. 




3. 이미 생성한 Next.js 13 프로젝트와 방금 생성한 repository를 연결합니다.




4. repository에 push가 제대로 이뤄졌다면 다음과 같은 문구가 나옵니다.











  • Github repository Setting



1. 생성한 프로젝트 좌측 메뉴에 Pages를 클릭합니다.





2. Pages에서 Build and deployment 하단에 있는 Source를 클릭합니다.




3. GitHub Actions를 클릭합니다.





4. gif와 같이 configure 설정 후 commit changes를 클릭하고 생성하면 git action이 생성됩니다.

이제부터는 github main 브랜치에 커밋할 때마다, git action에서 build됩니다.
